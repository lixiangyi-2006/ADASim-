# ADASim – 轻量级自动驾驶仿真平台
## 项目简介
**ADASim** 是一个基于 C++/Qt 与 Python 的自动驾驶仿真平台，采用算绘分离架构。  
本项目基于开源的ADASim框架进行系统级的深度优化与重构，原框架实现了基础的仿真闭环，我在此基础上正对嵌入式linux环境，进行增强，包括**性能瓶颈定位 内存管理优化 实时性改进 资源监控 网络健壮性加固等**使其能够高效运行在资源受限的 ARM 设备上。
## 项目使用运行方法
### 环境要求
- 操作系统：Ubuntu 20.04 / 22.04（或其他支持 Qt 的 Linux 发行版）
- 编译器：g++ 9+（支持 C++17）
- Qt 版本：5.15 或 6.x（Core, Gui, Widgets, Network, Sql）
- Python 3（运行规控节点时需要）
## 调试与性能分析
### 本项目开发中系统使用了以下工具：
- GDB：断点调试、多线程栈回溯

## 调试修改记录
### 物理引擎时间同步（解耦帧率与物理速度）
#### 原始问题
车辆位移固定为每帧 0.5 米，速度计算使用硬编码 dt = 0.1 秒。修改定时器间隔后，实际移动速度会变化，但仪表盘速度显示仍为固定值，物理引擎与帧率强耦合，不符合真实物理规律。
#### 调试：
1.用 std::chrono::steady_clock 计算真实时间差 dt，并传递给 MainWindow。
2.位移计算改为 deltaX = TARGET_SPEED * dt，速度计算使用真实 dt 替换硬编码 0.1。
3.提取回放模式下的占位 dt 和速度滤波系数为常量。
#### 效果
无论定时器间隔设为 100ms、200ms 还是 500ms，车辆实际移动速度和仪表盘显示速度均保持 5 m/s不变，物理引擎与帧率彻底解耦。
### 环形缓冲区替换历史帧队列
#### 原始问题
DataLoader 使用 QVector<FrameState> 存储历史帧，超过容量时调用 pop_front()，每次会移动所有剩余元素（O(N)），在帧数较大时（当前测试是5000帧以上）产生显著的 CPU 开销和内存移动。
#### 调试
1.实现模板类 RingBuffer<T>（固定数组 + 头尾指针，O(1) 写入与访问）。
2.替换 QVector<FrameState> 为 RingBuffer<FrameState>，写入改为 push()，自动覆盖最旧数据。
#### 效果
在 5000 帧压力测试下，原 QVector 版本因 pop_front() 产生大量 memmove 指令，环形缓冲区指令数减少约 90%，拖拽进度条、长时间运行更平滑。
### 增加了系统性能监控面板
#### 原始问题
程序运行时无法直观了解 CPU、内存、线程数等资源占用
#### 调试
在状态栏添加 CPU、内存、线程数标签，使用 QTimer 每秒更新。
#### 效果
状态栏右下方，绿色小字实时显示 CPU、内存、线程数标签
### 修改了下配置系统
#### 原始问题
配置文件缺失时加载失败，且所有参数必须由用户完整提供，缺少字段会导致 config_ 中缺失对应 key，调用者需手动处理默认值，代码冗余且无内置默认值。
#### 调试
1.定义内置默认配置，文件不存在时自动使用。
2.实现用户配置与默认配置合并
3.使用 QReadWriteLock 保证线程安全，预留 save() 和 configChanged 扩展接口。
#### 效果
零配置运行，首次启动无需配置文件；配置文件简化
### 仿真模拟：雷达点云噪声与运动畸变模拟
#### 原始问题
仿真点云为完美圆形且每帧相同，过于理想，无法测试感知算法对真实雷达噪声和运动畸变的鲁棒性
#### 调试
1.使用 std::normal_distribution 为点云添加高斯噪声（标准差 0.05 m）。
2.根据车速给点的 x 坐标增加与速度成正比的畸变偏移。
#### 效果
点云从完美的圆形变为带有毛刺、轻微拉伸的圆环，更贴近真实激光雷达，帮助提前发现感知算法对噪声或畸变的鲁棒性问题
### 完善TCP，加了心跳检测与断线重连
#### 原始问题
原 SocketServer 仅依赖客户端主动发送 disconnected 信号清理资源。若客户端异常断电、网络闪断或进程崩溃，服务器无法感知，导致僵尸连接堆积、端口泄漏。
#### 调试
1.为每个客户端创建心跳定时器（5 秒），发送 {"type":"PING"}；Python 端回复 {"type":"PONG"}。
2.维护 missedPongs 计数器，连续 3 次未收到 PONG 则主动断开并清理资源。
#### 效果
正常运行时定期 PING/PONG 日志；异常断连后 15 秒内自动断开僵尸连接，端口资源释放，连接计数归零。

**功能**：用 KD-Tree 将聚类邻域搜索从 O(N²) 降至 O(N log N)，提升点云处理性能。
